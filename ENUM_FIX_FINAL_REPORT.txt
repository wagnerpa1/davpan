🎯 PROBLEM
──────────────────────────────────────────────────────────────────────────────

Fehler:     "operator does not exist: text <> tour_status"
Root Cause: TEXT-Variablen wurden mit Enum-Typen verglichen
Impact:     PostgreSQL konnte Operationen nicht durchführen
Severity:   🔴 KRITISCH (System nicht funktionsfähig)

Betroffene Funktionen:
  • sync_tour_status_explicit()
  • register_tour_atomic()
  • promote_from_waitlist()


✅ LÖSUNG
──────────────────────────────────────────────────────────────────────────────

Implementiert 3 Hauptfixes:

1. VARIABLE TYPES
   ❌ DECLARE v_status TEXT;
   ✅ DECLARE v_status public.tour_status;

2. EXPLICIT CASTS
   ❌ WHERE status = 'confirmed'
   ✅ WHERE status = 'confirmed'::public.participant_status

3. NULL-SAFE COMPARISONS
   ❌ IF old != new
   ✅ IF old IS DISTINCT FROM new
